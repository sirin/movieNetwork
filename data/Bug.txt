programı yazarken gözden kaçırılan, unutulan ufacık şeylerden ortaya çıkan kocaman yanlışlardır. bir bug illaki programın sıçmasına sebep olmayabilir. görünürde normal çalıştığı halde esasen yanlış çalışıyor olabilir ki bunun tespiti diğerine göre bir miktar daha uğraş ister. hele kodunuzda race condition mevcut ise sıçtınız gitti. çözmek için debugging ile sorunlu bölümü tespit etme yoluna gidilir. çoğu zaman bu yöntemle bile bug'ın ortaya çıkması zaman alabilir zira kodunuz gözle görüldüğünde oldukça normal görünebilir. böyle durumlarda compiler'ınızın uyarılarını lütfen göz ardı etmeyiniz. void do_something(dword val) { if (val < 0) { mutlakayapilmasilazim(); } amanaman(); } do_something(-500); veya if (read(&data,0,sizeof(mydata) == sizeof(mydata))) { printf("mydata struct loaded!"); } resimdeki hataları bulunuz.
